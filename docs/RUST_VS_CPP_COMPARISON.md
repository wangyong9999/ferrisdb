## 锁架构对比

| 机制 | FerrisDB (Rust) | dstore (C++) |
|:-----|:---|:---|
| **状态字修改** | **纯 CAS** (无中间态) | fetch_add + CAS 回滚 |
| **等待策略** | parking_lot FIFO 队列 | OS semaphore |
| **写者保护** | DISALLOW_PREEMPT | DISALLOW_PREEMPT |
| **BTree 搜索锁** | Shared lock 下降 | Shared lock 下降 |
| **BTree 分裂锁** | **页级锁** (无全局锁) | 页级锁 |
| **Buffer 分配锁** | **锁外 flush** (I/O 不阻塞) | 锁内 flush |

**Rust 改进点**：CAS-only 状态修改消除了 C++ 的 fetch_add/sub 竞态窗口 (已验证 60/60 通过)
